About

Recall For Faster Incident Fixes

Recall is an on-call intelligence workspace that learns from already-resolved incidents and the code changes that fixed them. It helps teams avoid repeating investigation work and move from alert to proven fix faster.

How To Use (Most Important)

Use Recall In 4 Steps

This is the fastest way to get high-quality recommendations from your own incident history.

1

Integrate Organization

Open Control Center, add Azure DevOps/Jira profile, and save the active workspace.

2

Sync Resolved Tasks

3

What And Why

What We Do, Why We Need Recall

What Recall Does

Finds similar past incidents, maps them to known fixes, and returns ranked patch recommendations with reasoning.

Why Teams Need It

On-call time is expensive. Recall reduces repetitive debugging and helps engineers reuse proven fixes confidently.

Why It Is Better For On-call

Recommendations are evidence-backed, not generic. Engineers see similar incidents, fix history, and confidence signals.

Operational Safety

Local-first storage and controlled organization integration keep rollout practical for real production teams.

Technology

What Runs Under The Hood

Recall combines retrieval, ranking, and code-context learning so recommendations are specific, explainable, and actionable.

Frontend: Plain JavaScript workspace with local-first persistence and fast interaction.
Ingestion: Azure DevOps and Jira profile-based sync for resolved incidents.
Retrieval: Hybrid BM25/semantic candidate generation plus structured signal matching.
Embeddings: Ollama local embedding models (for semantic recall and similarity).
Ranking: Weighted rerank pipeline with feedback boost and confidence-aware abstain.
Explainability: Similar incident evidence + matched signals shown with recommendations.
